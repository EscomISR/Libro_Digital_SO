1.5 Shell

Interfaz entre el usuario y el sistema operativo

El Shell es la interfaz entre el usuario y el sistema operativo, responsable de interpretar los comandos introducidos y traducirlos en llamadas al sistema que el kernel puede ejecutar.

Ahora bien también sirve como interprete donde todo las llamadas ala sistema que realice el usuario las administra y las procesa en donde gira instrucciones al sistema de que se den de ejecutar o detener, existe la posibilidad de tener una línea de comando o una interfaz gráfica, esto permitirá la automatización de tareas utilizando scripts para realizar copias de seguridad permitiendo tener tareas especificas que permitan administrar todos los procesos complejos del sistema y así manejar también volúmenes de datos en un orden de archivos y directorios.

1.5.1
Argumentos y variables de entorno

Son todos los datos que el sistema operativo va a utilizar para que pueda almacenar específicamente todas las configuraciones y preferencias de donde se encuentran los archivos o directorio, además de que acedan a los programas para validar su comportamiento, aun así los argumentos son todos aquellos que se involucran al pasar por medio de comandos a todo programa que se desee ejecutará esto les permitirá modificar su secuencia en la acción que se desea llevar acabo.

En los scripts de shell, los argumentos se representan mediante variables posicionales como $0 (nombre del script), $1, $2, etc., que permiten procesar información dinámica sin modificar el código base.

Por otra parte, las variables de entorno almacenan información global sobre el entorno del sistema y son heredadas por los procesos hijos al ejecutarse un programa. Algunas de las más comunes son:

  • PATH: define las rutas donde el sistema busca los ejecutables.
  • HOME: indica el directorio personal del usuario.
  • USER: contiene el nombre del usuario actual.
  • SHELL: especifica el intérprete de comandos por defecto.
  • PWD: almacena el directorio de trabajo actual.

Estas variables pueden visualizarse o modificarse mediante comandos como printenv, set, env, export o unset. Gracias a ellas, el shell adapta el comportamiento de los programas y facilita la comunicación entre el usuario y el sistema operativo.

“Las variables de entorno proporcionan un mecanismo para pasar información entre el entorno del usuario y los programas en ejecución, definiendo parámetros del sistema y preferencias del usuario”
(Silberschatz, Galvin y Gagne, 2022, p. 327).

Asimismo, los argumentos permiten reutilizar scripts y comandos, otorgando flexibilidad, personalización y eficiencia en la administración del sistema.

1.5.2
Estructuras de control (if, for, while)

Esto permitirá establecer la secuencia lógica en la cual se visualizará la ejecución de todas las líneas de comando para que con ellos se facilite la forma más adecuada para decidir qué es lo mejor para el sistema, es por ello que el shell establecerá las opciones más adecuadas para implementar los lenguajes de programación e incorporar condiciones propias para su ejecución.

1.5.3
Manipulación de cadenas y entradas

El manejo de cadenas y entradas en el shell posibilita que se procese y modifique el texto de manera directa, lo cual hace más sencillo automatizar tareas y gestionar datos con eficacia.

El flujo de datos entre procesos puede ser controlado con redirecciones (>, <, >>) y tuberías (|), mientras que el usuario tiene la posibilidad de buscar, reemplazar o dar formato a la información mediante comandos como sed, awk, tr, grep, cut y echo.

1.5.4
Funciones y scripts

Las funciones y scripts son partes muy importantes para el buen funcionamiento del shell ya que ellas se encargan de ser parte esencial de otorgar el máximo rendimiento a la administración del sistema, así como tener una mayor forma de utilizar el código ya generado y con ello poder tener mayor la automatización de las tareas que el sistema genera o realiza.

Ahora bien, una tarea en especifico que realiza el sistema es tener en ejecución un bloque de código el cual le permitirá utilizar la cantidad de veces dentro de los script para que con ello ayude a llevar a cabo la tarea asignada dentro del sistema, todo esto es utilizado con el fin de implementar rutinas como que ayuden a llevar respaldo de seguridad para el sistema.

Aspecto Funciones Scripts
Definición Bloques de código reutilizables dentro del shell que ejecutan tareas específicas. Archivos de texto que contienen una secuencia de comandos ejecutados en orden.
Alcance Se ejecutan dentro del entorno actual del shell, compartiendo variables y contexto. Se ejecutan como procesos independientes, con su propio entorno.
Propósito Modularizar código y evitar la repetición de instrucciones dentro de un script. Automatizar tareas completas o procesos del sistema.
Ejecución Se invocan por su nombre dentro del mismo script o sesión. Se ejecutan con bash nombre_script.sh o mediante cron.
Ventajas Facilitan la legibilidad, mantenimiento y reutilización del código. Permiten automatizar procesos complejos y programar tareas recurrentes.
Uso típico Repetición de tareas dentro de un mismo script o entorno. Administración, configuración y mantenimiento del sistema.
