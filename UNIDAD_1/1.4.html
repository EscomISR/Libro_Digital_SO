1.4 Llamadas a sistema

Interfaz entre aplicaciones y el sistema operativo

Es la forma en que las aplicaciones o programas que el usuario requiere hace la petición de algún proceso al sistema operativo, es por ello que actuara como una interfaz para que están puedan acceder a los recursos y puedan con ello realizar las tareas solicitada, esto dependerá principalmente de los privilegios que se tengan en las llamadas ya que pueden ser desde la lectura de un archivo o hasta escribir en él y van a depender de la evaluación que el kernel realice a todas estas peticiones o llamadas del usuario al sistema.

“las llamadas al sistema representan el punto de transición entre los programas de usuario y el núcleo, proporcionando los medios mediante los cuales los procesos solicitan servicios del sistema operativo”
(Silberschatz, Galvin y Gagne, 2022, p. 140).

1.4.1
Procesos y señales

Representan dos de los mecanismos más importantes para la administración y comunicación dentro del sistema operativo.

Ambos elementos son gestionados directamente por el kernel, a través de un conjunto específico de llamadas al sistema que garantizan su control seguro y eficiente.

Estos se deben de realizar, aplicar, sincronizan y finalizan mediante la aplicación de las llamadas en donde el kernel evaluara su ejecución o función para llevarla a cabo.

  • Este comando forma nuevos procesos llamado Fork ().
  • execve(): reemplaza el espacio de memoria del proceso actual por un nuevo programa.
  • wait() y waitpid(): permiten que un proceso padre espere la finalización de sus hijos.
  • Este comando termina cualquier proceso y se escribe como exit ()

Cada una de estas llamadas activa rutinas del planificador de procesos (scheduler), encargado de asignar el tiempo de CPU y administrar el cambio de contexto entre procesos. Gracias a estas operaciones, el sistema operativo puede mantener múltiples tareas activas de forma concurrente, distribuyendo los recursos según políticas de prioridad y equidad.

“el kernel administra los procesos a través de un conjunto de llamadas al sistema que permiten su creación, suspensión, reanudación y finalización de manera controlada”
(Tanenbaum y Bos, 2015, p. 154).

Por otro lado, las señales constituyen un mecanismo de comunicación y control entre procesos o entre el sistema y un proceso específico. Son eventos asincrónicos generados por el kernel o por otros procesos para notificar sucesos importantes, como errores, interrupciones o solicitudes de terminación. Por ejemplo, señales como SIGINT interrumpen un proceso (generalmente mediante la combinación Ctrl + C), mientras que SIGKILL fuerza su terminación inmediata. Otras señales, como SIGCHLD, informan al proceso padre sobre la finalización de un hijo.

Las llamadas al sistema encargadas de manejar señales incluyen signal(), kill(), sigaction() y sigprocmask(), que permiten enviar, recibir y bloquear señales según el contexto.

1.4.2
Directorios y archivos

Estos elementos son de suma importancia en los sistemas operativos ya que los archivos son un grupo de datos que se identifican por un nombre y una extensión y un directorios es donde se organizan y se listan los archivos que tienen guardados dentro de esta estructura y que son fundamentales para el buen manejo tanto del usuario como del sistema operativo.

Existen diferentes tipos de archivos y la extensión establece el programa en donde fue creado, así como siempre representaran la capacidad que estos tengan en almacenamiento ocupado

Los directorios son de un tipo organización los archivos en donde actúa como contenedor donde los agrupa lógicamente relacionando su administración y su ubicación con ello permite que no se consuma recursos de hardware y memoria para su localización. Existe director que son organizados en forma jerárquica creando una estructura llamada árbol o raíz para mantener el orden en el sistema de archivos donde sea más eficiente su ubicación de los archivos.

1.4.3
Protección y control de acceso

Este proceso va a establecer los mecanismos necesario para que los usuarios que tengan privilegios puedan acceder a datos, aplicaciones y recursos mediante procedimientos bien específicos bajo contraseñas y accesos que el usuario tenga así como lista de control de acceso, los más autorizados son aquellos que tengan autentificación, que tenga acciones para el uso de la aplicación por el usuario realizando autorización para llevarlo a cabo, listas de control, control basado en roles, estado de mejor acceso dependiendo de su privilegio, contrafuegos y programas de seguridad y por último es muy importante sus actualizaciones para funcionar adecuadamente ante el sistema.

El control de acceso también se extiende al ámbito de la memoria y los dispositivos. El kernel utiliza tablas de páginas con permisos, asegurando que cada proceso acceda solo a las direcciones de memoria asignadas, y bloqueando operaciones de lectura o escritura no autorizadas. De manera similar, las llamadas relacionadas con la administración de archivos verifican los identificadores de usuario (UID) y grupos (GID) antes de ejecutar cualquier operación. Este modelo refuerza la seguridad en entornos multiusuario y protege los datos del sistema ante accesos indebidos.

1.4.4
Diversas (llamadas misceláneas)

Son un grupo de funciones, aplicaciones o herramientas en donde se visualizan los servicios esenciales del sistema por lo que no se relacionan con la memoria, procesos o archivos solo se usan para que todo funcione en forma general correctamente y tiene su función de interactuar con el usuario y su forma de comunicarse con el sistema realizando llamadas al sistema, interfaces de programación uso de bibliotecas, servicios de procesos en segundo plano, uso de herramientas e interacciones con el hardware.

Por lo que existen una categoría para las llamas que son:

  • Gestión del tiempo y fecha: time(), gettimeofday(), clock_gettime() permiten consultar y modificar la hora del sistema o medir intervalos temporales.
  • Gestión del entorno: getenv(), setenv() — acceden o modifican variables de entorno que influyen en el comportamiento de los procesos.
  • Gestión del sistema: uname() — obtiene información del sistema operativo (nombre, versión, arquitectura).
  • Sincronización: sleep(), usleep() — suspenden temporalmente la ejecución de un proceso.
  • Gestión de recursos: getrlimit(), setrlimit() — consultan o establecen límites de uso de recursos como CPU, memoria o archivos abiertos.
  • Depuración y monitoreo: ptrace() — permite observar y controlar la ejecución de otros procesos, útil para depuradores.

Estas llamadas desempeñan un papel complementario en la administración del sistema, proporcionando herramientas que permiten monitorizar, ajustar y sincronizar el funcionamiento del sistema operativo con las aplicaciones.
