Implementación de E/S Bloqueante (Wait/Signal)

Instrucciones:

Cuando un proceso hace una llamada al sistema bloqueante (como scanf), el SO debe suspenderlo. Completa la lógica del kernel para poner al proceso a DORMIR mientras espera el hardware, y para DESPERTARLO dentro de la interrupción.

Kernel_IO_Driver.c 

// Función del Kernel llamada por 'scanf' o 'read'
void sistema_leer_datos() {
    // 1. Iniciar la operación en el controlador de dispositivo
    iniciar_hardware_io();

    // 2. El dato no está listo. Debemos bloquear este proceso.
    printf("[KERNEL] Poniendo el proceso en espera...\n");

    // 3. Al despertar, copiamos el dato del buffer del kernel al usuario
    copiar_a_usuario();
}
