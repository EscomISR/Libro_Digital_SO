6.2 Virtualización

Conceptos fundamentales y tipologías

Técnica que permite ejecutar múltiples procesos aislados y simultáneos sobre un mismo hardware físico, mediante la creación de máquinas virtuales (VMs) que simulan los recursos del sistema.

Tipo de virtualización Componentes principales Ventajas Desventajas
Virtualización completa Hipervisor tipo 1 o tipo 2, máquinas virtuales (VMs), controladores virtuales y recursos físicos simulados.
  • Aislamiento total entre máquinas.
  • Permite ejecutar sistemas operativos diferentes en el mismo hardware.
  • Ideal para pruebas y entornos educativos.
  • Requiere gran cantidad de recursos.
  • Sobrecarga de rendimiento por la emulación del hardware.
Paravirtualización Hipervisor, sistemas invitados modificados, controladores para virtualizados.
  • Mayor eficiencia al comunicarse directamente con el hipervisor.
  • Menor sobrecarga de hardware.
  • Requiere modificar el sistema operativo invitado.
  • No todos los sistemas son compatibles.
Virtualización a nivel de sistema operativo (contenedores) Núcleo compartido, entornos aislados (contenedores), gestor de contenedores (Docker, LXC).
  • Ligereza y rapidez en la ejecución.
  • Uso eficiente de recursos.
  • Ideal para microservicios y despliegues en la nube.
  • No permite diferentes núcleos de SO.
  • Menor aislamiento que las VMs tradicionales.
Virtualización asistida por hardware CPU con soporte de virtualización (Intel VT-x, AMD-V), hipervisor optimizado, dispositivos virtuales.
  • Aceleración de rendimiento.
  • Mejor soporte para sistemas modernos.
  • Reducción de la sobrecarga de emulación.
  • Depende del soporte del hardware.
  • Puede requerir configuración avanzada de BIOS o firmware.

6.2.1
Emulación

INTRODUCCIÓN

Técnica que utiliza un sistema que mide el comportamiento de otro sistema distinto, permitiendo que el software utilice su arquitectura específica para que funcione en diferentes plataformas utilizando los recursos de sistema en forma segura y eficiente.

Tipo de emulación Descripción Ejemplo
Emulación de hardware Reproduce el comportamiento de una arquitectura física (CPU, memoria, dispositivos). QEMU, Bochs.
Emulación de software o API Permite ejecutar aplicaciones de un sistema operativo en otro. Wine (Windows en Linux).
Emulación de sistema completo Replica todo el entorno operativo, incluyendo BIOS, kernel y periféricos. Virtual PC, MAME.
Emulación parcial o de instrucción Traduce solo ciertas funciones o llamadas al sistema. Emuladores de consolas o compiladores cruzados.
Tabla 11. Tipos de emulación. Adaptado de Conceptos de sistemas operativos (10.ª ed.) por A. Silberschatz, P. B. Galvin y G. Gagne, 2022, Wiley.

APLICACIONES DE LA EMULACIÓN

  • Compatibilidad de software antiguo: ejecutar programas de arquitecturas o sistemas obsoletos.
  • Desarrollo multiplataforma: probar software en distintos entornos sin hardware adicional.
  • Educación y simulación: recrear arquitecturas para prácticas académicas o investigación.
  • Seguridad informática: aislar software potencialmente peligroso en entornos emulados.

VENTAJAS

  • Facilita la migración de sistemas y aplicaciones.
  • Permite el aislamiento de pruebas sin riesgo para el sistema principal.
  • Útil para depuración y análisis forense.

DESVENTAJAS

  • Reducción del rendimiento por la traducción de instrucciones.
  • Limitaciones en el acceso a hardware especializado.
  • Complejidad en la configuración y sincronización del entorno.
  • Puede no reproducir con exactitud el comportamiento temporal del sistema real.

DIFERENCIAS ENTRE EMULACIÓN Y VIRTUALIZACIÓN

Aspecto Emulación Virtualización
Nivel de simulación Reproduce completamente otro hardware o arquitectura. Comparte directamente los recursos del hardware físico.
Compatibilidad Permite ejecutar sistemas de distintas arquitecturas (x86, ARM, etc.). Requiere arquitecturas compatibles.
Rendimiento Menor, por traducción de instrucciones. Mayor, al usar recursos nativos.
Uso principal Compatibilidad y preservación. Consolidación y administración de sistemas.

6.2.2
Virtualización Asistida por Hardware

INTRODUCCIÓN

Es una forma que utilizan los procesadores modernos para hacer más rápido la ejecución en entornos virtuales, con esto se reduce la sobre carga de la CPU.

FUNCIONAMIENTO

Conjunto de instrucciones que utiliza el procesador para lleva a cabo modos de operación para ejecutar múltiples procesos en paralelo con el sistema operativo.

Fabricante Tecnología Descripción
Intel VT-x (Tecnología de virtualización) Introduce nuevos modos de ejecución para máquinas virtuales y mejora la gestión de memoria mediante EPT (Extended Page Tables).
AMD AMD-V (AMD Vvirtualización) Permite ejecutar instrucciones privilegiadas directamente, e implementa Indexación rápida de virtualización (RVI) para optimizar la paginación.
ARM ARM Extensiones de virtualización Diseñadas para entornos móviles y embebidos, permiten virtualizar dispositivos en plataformas ligeras.
Tabla 13. Extensiones más comunes. Adaptado de Conceptos de sistemas operativos (10.ª ed.) por A. Silberschatz, P. B. Galvin y G. Gagne, 2022, Wiley.

VENTAJAS

  • Mayor eficiencia en la gestión de memoria: uso de tablas extendidas (EPT o NPT) que evitan múltiples niveles de traducción.
  • Aislamiento y seguridad mejorados: separación clara entre las máquinas virtuales y el sistema anfitrión.
  • Compatibilidad con hipervisores modernos: soporte nativo para KVM, VMware, Hyper-V, entre otros.

DESVENTAJAS

  • Configuración avanzada: puede requerir habilitar opciones en BIOS/UEFI.
  • Vulnerabilidades específicas: ataques como VM escape pueden afectar la capa de hardware virtualizado.

IMPORTANCIA EN LOS SISTEMAS OPERATIVOS MODERNOS

La virtualización asistida por hardware es esencial para:

  • Centros de datos y computación en la nube.
  • Entornos de servidores virtualizados.
  • Pruebas y desarrollo de software multiplataforma.

6.2.3
Paravirtualización

INTRODUCCIÓN

Técnica en la que el sistema operativo permite mejorar el rendimiento y eficiencia de los procesos virtuales permitiendo comunicación optimizada entre los interna y su entorno físico.

COMPONENTES PRINCIPALES

  • Hipervisor paravirtualizado: software central que coordina el acceso de los sistemas invitados a los recursos físicos (CPU, memoria, E/S).
  • Sistemas operativos invitados modificados: adaptados para comunicarse con el hipervisor mediante hypercalls.
  • Interfaces para virtualizadas (APIs): definen las operaciones de comunicación entre el guest y el host.
  • Controladores para virtualizados: optimizan la transferencia de datos en dispositivos de red y almacenamiento.

VENTAJAS

  • Mayor rendimiento: al evitar la emulación completa del hardware.
  • Menor latencia en operaciones de E/S.
  • Menor uso de CPU y memoria en el hipervisor.
  • Aislamiento y estabilidad entre máquinas virtuales.
  • Ideal para entornos con múltiples sistemas operativos similares.

DESVENTAJAS

  • Requiere modificar los sistemas operativos invitados, lo que limita la compatibilidad.
  • No puede ejecutar sistemas sin soporte de paravirtualización.
  • Depende de APIs específicas del hipervisor.
  • Menor flexibilidad que la virtualización completa.

COMPARACIÓN CON OTROS MÉTODOS

La tabla presenta una comparación entre la virtualización completa y la paravirtualización, destacando las diferencias clave en compatibilidad, rendimiento y manejo de recursos del sistema operativo.

Característica Virtualización completa Paravirtualización
Compatibilidad Alta, ejecuta sistemas sin modificaciones. Media, requiere SO adaptado.
Rendimiento Moderado, depende de la emulación. Alto, casi nativo.
Sobrecarga del hipervisor Alta. Baja.
Uso de hardware Emulado. Acceso directo controlado.
Ejemplo VMware, VirtualBox. Xen, KVM con drivers PV.

6.2.4
Contenedores

Son aquellos que representan una evolución de la virtualización moderna enfocada principalmente a estabilidad, portabilidad y crecimiento al compartir operaciones con el kernel para que con ello se evite la sobre carga en las máquinas virtuales teniendo como tal una solución al desarrollo de aplicaciones dinámicas.
