Sección Crítica con Mutex

Instrucciones:

Para evitar condiciones de carrera cuando varios hilos acceden a saldo_bancario, debemos usar un Mutex. Completa el código: primero bloquea (lock) el mutex antes de entrar a la sección crítica y luego desbloquéalo (unlock) al salir.

Mutex.c (Incompleto) 
void* realizar_transaccion(void* arg) {
    // ...
    printf("Hilo %d intentando entrar a la sección crítica...\n", id);

    // [DESAFÍO 1] Bloquea el mutex 'candado'
    // Función: pthread_mutex_lock(referencia)
    // --- INICIO SECCIÓN CRÍTICA ---
    saldo_bancario += monto;
    printf("Nuevo saldo: %d\n", saldo_bancario);
    // --- FIN SECCIÓN CRÍTICA ---

    // [DESAFÍO 2] Desbloquea el mutex 'candado'
    // Función: pthread_mutex_unlock(referencia)
    return NULL;
}
¡Excelente! Has asegurado la sección crítica correctamente. pthread_mutex_lock detiene a otros hilos si el candado está ocupado, y pthread_mutex_unlock libera el recurso para el siguiente hilo en espera.
Incorrecto. Revisa las funciones POSIX para mutex.
  • Para entrar: pthread_mutex_lock(&candado);
  • Para salir: pthread_mutex_unlock(&candado);
